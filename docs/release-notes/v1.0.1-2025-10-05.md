## UX Changes
- **Section Skipping:** Form navigation no longer checks required questions before advancing. Users can freely move between sections and only need to resolve validation errors when submitting. (`DynamicFormNavigation.tsx`)

## Developer Experience
- **Lint Hygiene:** Addressed unused variables, environment flag mismatches, and unescaped entities so `npm run lint` completes without warnings.
- **Shared Visibility Mode (2025-10-07 update):** Draft and submission lists now default to "show all" using a shared test identity (`shared-user`). Browsers no longer persist a per-tab session ID, which keeps manual reviewers in sync while a single basic-auth credential is in place. A new optional env var `NEXT_PUBLIC_TEST_USER_ID` can override the label.

## Security
- **Fail-Closed Basic Auth:** The Next.js middleware now returns HTTP 500 if either `BASIC_AUTH_USERNAME` or `BASIC_AUTH_PASSWORD` is unset, preventing the app from running without credentials. (`middleware.ts`)
- **Sanitized Logging:** Introduced a global logger that suppresses sensitive payloads in production while keeping developer-friendly logs locally.
- **Secrets hygiene:** Updated App Service `DATABASE_URL` / `PRISMA_MIGRATE_DATABASE_URL` with the rotated `triageadmin` credential. See `tech-triage-platform/.env.azurestudio.local` for the maintained connection string reference.
