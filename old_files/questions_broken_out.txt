Section 0. Header and identifiers

F0.1 Technology ID
Type: short text, required

F0.2 Invention Title
Type: short text, required

F0.3 Reviewer
Type: short text, required

F0.4 Date
Type: date, required

F0.5 Inventor(s)
Type: long text, required

F0.6 Department(s)
Type: long text, required

F0.7 Domain or Asset Class
Type: single select, required
Options: Medical Device, Therapeutic, Diagnostic, Digital Health, Research Tool, Other

Section 1. Technology overview and readiness

F1.1.a Provide a concise summary of the technology and the problem it solves
Type: long text, required

F1.1.b Identify the primary target users
Type: long text

F1.1.c Describe how it works at a high level
Type: long text

F1.1.d List the licensable asset type(s)
Type: multiselect
Options: Composition of Matter, Method, Device, Software Algorithm, Database, Trademark, Copyright, Other

F1.2.a Current stage of development
Type: single select, required
Options: Idea only, Lab proof of concept, Prototype, Validated prototype, Pilot in humans, On market

F1.2.b Has it been reduced to practice
Type: single select
Options: Yes, No, Partially, Unsure

F1.2.c Is there proof-of-concept data
Type: single select
Options: Yes, No, In progress

F1.2.d Is there a working prototype
Type: single select
Options: Yes, No, In progress

F1.2.e Manufacturing feasibility known
Type: single select
Options: Yes, No, Not applicable

Section 2. Strategic alignment and unmet need

F2.1.a Describe alignment with institutional mission or strategic goals
Type: long text, required

F2.1.score Mission Alignment score
Type: integer 0–3
Help text: 0 not aligned, 1 aligns with one dimension, 2 aligns with two, 3 aligns with all

F2.2.a Describe the unmet clinical or market need and potential impact
Type: long text, required

F2.2.b For diagnostics, state the clinical utility
Type: long text, show only if Domain includes Diagnostic

F2.2.score Unmet Need score
Type: integer 0–3
Help text: use late-stage competitor count and differentiation to anchor the score

Section 3. State of the art and IP position

F3.1.a Summarize relevant prior art and publications
Type: long text, required

F3.1.b Explain differentiators and why novel and non-obvious
Type: long text, required

F3.1.c For digital or software, note open source use, key algorithms or UI or databases, and protectability beyond patents
Type: long text, show if Domain includes Digital Health or Software

F3.2.a Assess IP protectability and claim strength
Type: long text, required

F3.2.b Is infringement detectable
Type: single select
Options: Yes, No, Unclear

F3.2.c Ease of design-around
Type: single select
Options: Hard, Moderate, Easy, Unclear

F3.2.d Any public disclosures that may limit patentability
Type: long text

F3.2.score IP Strength and Protectability score
Type: integer 0–3

Section 4. Market and commercial viability

F4.1.a Target market overview with TAM, patient counts, key trends, regulatory path, and reimbursement notes
Type: long text, required
Validation: allow hyperlinks

F4.2.a Competitive landscape table
Type: repeatable group with rows
Row fields: Company (short text), Product or Solution (short text), Description and Key Features (long text), Revenue or Market Share estimate (short text)

F4.3.a Stakeholders
Type: checkbox group
Options: Payers, Providers, Patients, Policymakers, Employers, Health systems, Other

F4.3.b Value proposition for each selected stakeholder
Type: long text, required if any stakeholder selected

F4.4.a Market Size score
Type: integer 0–3
Help text: use the scoring table by asset class

F4.4.b Patient Population or Procedural Volume score
Type: integer 0–3
Help text: 0 <200k, 1 is 200k–1M, 2 is 1M–2.5M, 3 is >2.5M

F4.4.c Number of Competitors score
Type: integer 0–3
Help text: 0 is >15, 1 is 11–15, 2 is 5–10, 3 is <5

Section 5. Inventor and path forward

F5.1.a Inventor engagement and credibility assessment
Type: long text, required

F5.1.b Willingness to participate in commercialization
Type: single select
Options: High, Moderate, Low, Unknown

F5.1.c Expectation realism
Type: single select
Options: Realistic, Needs guidance, Unrealistic, Unknown

F5.2.a Recommended commercialization pathway
Type: single select, required
Options: License to established company, Startup formation, Internal development via sponsored research, Not ready, Other

F5.2.b Rationale for pathway recommendation
Type: long text, required

Section 6. Scorecard and recommendation

F6.1 Summary of assessment
Type: long text, required

F6.2 Final recommendation
Type: single select, required
Options: Proceed, Consider alternative pathway, Close

F6.3 If not Proceed, state what data or development could change the recommendation
Type: long text, conditional on F6.2 not equal to Proceed

F6.4 Subject matter experts to consult
Type: repeatable group
Row fields: Name, Expertise, Contact information
